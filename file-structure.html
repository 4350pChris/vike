File Structure

Read the React Tour or Vue Tour, or play with the boilerplate $ npm init vite-plugin-ssr@latest, to get familiar with the .page.js and _default.page.{server|client}.js files.

Recommended file structure:

# Landing page (URL: `/`)
/pages/index/index.page.js
/pages/index/SomeComponentForLandingPage.js
/pages/index/**/* # More files specific to the landing page
# About page (URL: `/about`)
/pages/about/index.page.js
/pages/about/SomeComponentForAboutPage.js
/pages/about/**/* # More files specific to the about page
# We create a new directory `/pages/new/` for each page. I.e. we create
# `/pages/new/index.page.js` instead of `/pages/new.page.js`.
/pages/*/*.page.js

# (React/Vue/...) components shared by several pages
/components/

# Infrastructure code that specifies how our pages are rendered
/renderer/_default.page.server.js
/renderer/_default.page.client.js
/renderer/_error.page.js
/renderer/PageWrapper.js  # The root (React/Vue/...) component
/renderer/PageWrapper.css # that wraps the `Page` component defined in `/pages/**/*.page.js`.
/renderer/Header.js # Our website header
/renderer/logo.svg # Our website logo

# Express.js/Fastify/... code
/server/

That said, we can define .page.js files wherever we want and we can use whatever file structure we want.

For domain-driven file structure, like the following, write a comment at GitHub - brillout/vite-plugin-ssr - #125.

# Marketing pages
/marketing/pages/index/index.page.js
/marketing/pages/about/index.page.js
/marketing/pages/jobs/index.page.js
/marketing/_default.page.route.js

# Auth related code & pages
/auth/pages/signup/index.page.js
