`/pages/+*.js` files (directly under `/pages`) allow us to define hooks and settings that apply to all pages.

For example, we usually define an <Link text={<><code>onRenderHtml()</code> hook</>} href="/onRenderHtml" /> in
`+onRenderHtml.js` in order to define how our pages are rendered to HTML.

```js
// /pages/+onRenderHtml.js
// Environment: server

export { onRenderHtml }

// This `onRenderHtml()` hook applies to all our pages
async function onRenderHtml(pageContext) {
  return escapeInject`<!DOCTYPE html>
    <html>
      <head>
        <title>My SSR App</title>
      </head>
      <body>
        <div id="root">${dangerouslySkipEscape(renderToHtml(pageContext.Page))}</div>
      </body>
    </html>`
}
```

This is equivalent to defining the `onRenderHtml()` hook in `/renderer/+onRenderHtml.js`, is however useful if we don't own
the `/renderer` directory, e.g. if it is located in a third-party package like the
<Link text={<><code>vike-*</code> packages</>} href="/vike-packages" />.


## Multiple defaults

There can be multiple defaults:

```
marketing/pages/+onRenderHtml.js
marketing/pages/+onRenderClient.js
marketing/pages/index/+Page.js
marketing/pages/about/+Page.js
marketing/pages/jobs/+Page.js

admin-panel/pages/+onRenderHtml.js
admin-panel/pages/+onRenderClient.js
admin-panel/pages/index/+Page.js
```

The `marketing/+*.js` files apply as default to `marketing/**/+Page.js`, while `admin-panel/+*.js` apply as default to
`admin-panel/**/+Page.js`.
